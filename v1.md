# Ecosystem + Evolution Simulation — v1 Implementation Spec

> **Purpose of this file:** a *self-contained* spec for implementing v1. A fresh session
> with no prior context should be able to build the whole thing from this document alone.
> No project code exists yet — this is a clean slate. **No neural networks in v1.**
> Read top to bottom before writing any code.

---

## 0. The one-sentence summary

Build a headless, deterministic ecosystem simulation (plants / sheep / fox on a
noise-generated world with biomes, water, weather and seasons) where entities act via
**hardcoded rules behind a `brain.decide(observation) -> action` contract**, carry a
**heritable genome**, and **perceive only their local surroundings** — so that evolution
and predator–prey dynamics emerge and can be measured. The hardcoded brain is throwaway;
the *architecture around it* is built to later swap in PyTorch neural brains with zero
sim rewrite.

---

## 1. Project goal

Study **evolution, ecosystem dynamics, and entity interactions** through a simulation.
The research output is data: population curves, predator–prey oscillations, and trait
drift over generations.

This is **v1** of a long-term project. The long-term endgame (NOT in v1, but the reason
for several design choices below) is fully neural, learning, and *evolvable-morphology*
brains in PyTorch. v1 must not paint itself into a corner that forces a rewrite.

## 3. Architecture principles (non-negotiable — these protect the long-term vision)

1. **Headless sim, separate renderer.** The `sim/` core is headless, deterministic
   (seeded RNG), and runnable far faster than real time. The Arcade renderer is an
   optional *observer* attached only when watching. `run_live.py` and
   `run_experiment.py` share the exact same `sim/`. **`sim/` must never import `render/`.**

2. **The brain↔world contract is the spine.** Every entity decision flows through
   `brain.decide(observation) -> action` plus a reward/fitness signal. v1 uses a
   rule-based brain behind this *same* contract, so dropping in a PyTorch brain later is a
   one-file change. The real design work of v1 is the **per-species observation (egocentric
   grids + scalars) and action schemas** (§7.2–7.3), not the throwaway rules.

3. **The BrainSystem is global and batched.** It collects every agent's observation (grouped
   by species) → produces every action → scatters them back into the global ordering.
   Pointless for rules; built now so the neural version is a drop-in. Never make `decide` a
   per-agent method on an entity.

## 5. Key decisions made (with rationale — flip if you disagree, but know why)

- **DATA LAYOUT: Structure-of-Arrays (SoA) in NumPy.** Entity state lives in parallel
  arrays indexed by slot (`pos[i]`, `energy[i]`, `genome[i]`, …), *not* one object per
  entity. **Rationale:** the batched brain wants observations as a matrix anyway; local
  perception runs as array radius-queries off the spatial grid; scales to thousands; zero
  refactor when neural brains arrive. Cost: less beginner-friendly than objects. *(If you
  strongly prefer readability while learning the domain, objects-now / arrays-later is a
  legitimate fallback — but it's a known future refactor.)*

- **WATER: freshwater (rivers + lakes) is drinkable; ocean is a saltwater barrier**
  (impassable, not drinkable). **Rationale:** makes inland freshwater the limiting
  resource so animals cluster near rivers — interesting spatial dynamics, and gives rivers
  a real purpose.

- **PERCEPTION: local radius only.** No entity ever queries a global "nearest". It sees
  food/threats/mates only within its **sensory range** (a heritable trait). Beyond it,
  blind. **Rationale:** global knowledge is omniscience and contradicts reality; local-only
  perception is what *creates the need* for memory later, making the future LSTM/memory
  phase meaningful instead of redundant.

- **REPRODUCTION: sexual.** Two parents (opposite sex, same species, adult, healthy, in
  proximity) → offspring genome = crossover + mutation. **Rationale:** richer, more
  realistic evolutionary model than asexual cloning; proximity requirement ties directly
  into local perception.

- **EXPLORATION is load-bearing.** Because senses are local, animals spend much time
  perceiving nothing; the "wander" branch must be real (directed walk with momentum), not
  a throwaway. In v1 blind = explore; later, memory turns blind-time into informed travel.

## 7. The spine

### 7.1 Brain↔world contract

```
# brain.py (interface)
class Brain:
    def decide(self, obs_by_species, idx) -> np.ndarray:
        """obs_by_species: {species_id: Observation} (per-species egocentric grids +
        scalars, §7.2);  idx: global alive slot ids.
        -> actions: (len(idx), ACT_DIM) float32, aligned to the GLOBAL ordering idx."""
        raise NotImplementedError

class RuleBrain(Brain):
    # v1: vectorized hardcoded priority logic (§13). No learning, no state. Because it
    # can't convolve, it first DECODES each species' grid channels into nearest/best
    # targets; a neural brain learns straight off obs.grids.
    ...
```

`BrainSystem` (in `systems/brain_system.py`) is the only caller: it asks `perception`
for the per-species observations, calls `brain.decide`, and hands the
`(len(idx), ACT_DIM)` action matrix (aligned to the global alive ordering) to the
movement/consumption/reproduction systems. Swapping `RuleBrain` for a `TorchBrain` later
changes nothing else — the grids are already a CNN's channel-stack.

### 7.2 Observation schema (LOCAL, egocentric **grids** + scalars, **per-species**)

Perception is **separated by species**, and each agent's external view is a stack of
**egocentric grids** (not a flat nearest-target vector). The grids are the raw local view —
exactly a CNN's channel-stack, which is the whole point of the design. `perception.build`
returns `(obs_by_species, idx)`: a `{species_id: Observation}` dict plus the **global** alive
ordering `idx`. Each `Observation` carries:

- **`grids`** `(N, C, K, K)` float32 — `C` egocentric channels per agent, a window of side
  `K = 2·R + 1` centred on the agent, where `R = ceil(largest sensory_range across all
  species)` (one fixed canvas for batching). Cells beyond the agent's **own** heritable
  `sensory_range` (a circular eye-mask) or off-world are **zeroed** — local-only, no global
  "nearest".
- **`scalars`** `(N, SCALAR_DIM)` float32 — internal state + global env (below).
- **`radius`** `R`, **`idx`** (global slot ids of its rows, for scatter-back), **`species`**.

**Grid channels (per species — each carries only what it uses, so a CNN has no dead inputs):**

| Species | Channels (index order) |
|---|---|
| **Sheep** (5) | `terrain`, `water`, `food` (= grass field), `threat` (= foxes), `mate` |
| **Fox** (4) | `terrain`, `water`, `food` (= exposed prey), `mate` |

The `food` channel is unified in position but species-specific in content (vegetation for the
herbivore, prey entities for the carnivore). `terrain`/`water` are windowed static fields;
`food`/`threat`/`mate` mark in-range cells/entities (prey hidden in cover are invisible to
foxes; mate = opposite-sex adult conspecific; the threat scatter is skipped when no foxes
exist; juveniles skip the mate query). Foxes have no predators → no `threat` channel.

**Scalars (identical layout for both species), `SCALAR_DIM = 11`:**

| idx | field | encoding / range |
|---|---|---|
| 0 | hunger | [0,1] |
| 1 | thirst | [0,1] |
| 2 | energy | [0,1] |
| 3 | health | [0,1] |
| 4 | age | age / max_age, [0,1] |
| 5 | sex | {0,1} |
| 6 | local temperature (own cell) | normalized |
| 7 | time of day | [0,1] (0=midnight) |
| 8 | season | [0,1] (0=start of year) |
| 9 | sensory_range | cells (for distance normalization + the CNN) |
| 10 | diet | {0,1} (1 = carnivore) |

### 7.3 Action schema

| idx | field | meaning |
|---|---|---|
| 0 | move_dx | desired heading x, [-1,1] (scaled by speed trait) |
| 1 | move_dy | desired heading y, [-1,1] |
| 2 | eat | gate [0,1]; act if > 0.5 and food adjacent |
| 3 | drink | gate [0,1]; act if > 0.5 and freshwater adjacent |
| 4 | reproduce | gate [0,1]; act if > 0.5 and mate adjacent + conditions met |

`ACT_DIM = 5`.

### 7.4 Tick order (fixed timestep)

Each `Simulation.step(dt)` runs systems in this exact, deterministic order:

```
1. environment.update(dt)     # advance clock; update season, weather, temperature offsets
2. grid.rebuild()             # spatial hashes (global + per-species) from positions
3. obs_by_species, idx = perception.build()   # per-species egocentric grids + scalars (LOCAL)
4. act = brain_system.decide(obs_by_species, idx)   # (len(idx), ACT_DIM), aligned to idx
5. movement.apply(act)        # headings -> positions; terrain speed cost; water blocks
6. consumption.apply(act)     # eat plants / fox eats sheep / drink water
7. metabolism.apply(dt)       # energy drain, hunger+, thirst+ (heat-scaled), health, aging, deaths
8. reproduction.apply(act)    # mate within range -> crossover+mutation -> spawn offspring
9. vegetation.grow(dt)        # plants regrow from nutrients/moisture/season
10. logger.record()           # stats for this tick
```

Determinism: iterate entities by index, two seeds — world seed for generation + a single
seeded run RNG for dynamics (§15) — fixed `dt`.

## 8. World generation (`world.py` + `hydrology.py`)

Generated once at startup from the **world seed** (terrain noise + hydrology only — the run
seed never touches generation, so a world seed always reproduces the same map; §15). Grid of
`WORLD_W × WORLD_H` cells.

### 8.1 Continuous fields

- **elevation**: layered OpenSimplex (several octaves, decreasing amplitude /
  increasing frequency), normalized to [0,1].
- **moisture**: independent OpenSimplex field, [0,1]. Boosted near freshwater (§8.2).
- **temperature (static part)**: `base_lat(y) − elevation * LAPSE_RATE`. `base_lat`
  gives a latitude gradient (one edge colder). The **dynamic** part (season + day/night
  global offsets) is added at runtime by `environment` (§9), so the field stored here is
  spatial-only.

### 8.2 Hydrology (ocean + rivers + lakes)

```
sea_level = SEA_LEVEL_THRESHOLD               # on normalized elevation

# Ocean: low cells connected to the map border (flood-fill from edges).
ocean = floodfill(elevation < sea_level, from=borders)
# Inland low cells not reached by ocean flood become candidate basins (lakes).

# Rivers: carve downhill from sources to the sea.
for source in pick_high_cells(N_RIVER_SOURCES):     # highest / random-high elevation
    pos = source
    while True:
        mark river[pos] = True
        nb = lowest_elevation_neighbor(pos)          # 8-neighborhood
        if ocean[pos] or river_already_here(nb): break
        if elevation[nb] >= elevation[pos]:          # local minimum
            lake = floodfill_basin(pos, up_to_spill_level)   # form a lake
            mark lake cells; pos = spill_cell or break
        else:
            pos = nb

beach = land cells within 1 step of ocean
freshwater = river | lake                            # DRINKABLE
water_any  = freshwater | ocean                      # ocean blocks movement (see §14)

# Moisture boost: cells near freshwater get +moisture with distance falloff
moisture += freshwater_distance_falloff()
```

Robustness note: the "stop at local minimum / make a small lake" rule is acceptable for
v1. The proper upgrade (if rivers look broken) is depression-filling via **Priority-Flood**
before carving, then D8 flow-accumulation thresholding — defer unless needed.

### 8.3 Biomes (Whittaker-style lookup, gradual transitions)

Biome is a *label* derived from continuous (temperature, moisture, elevation). Entities
read the continuous fields, so transitions are gradual automatically; the label is only
for **render color** and **base nutrient / plant suitability**.

Lookup priority:
```
if ocean -> OCEAN ; elif beach -> BEACH
elif elevation > MOUNTAIN_THRESHOLD -> MOUNTAIN
elif temperature < COLD_THRESHOLD -> COLD (tundra/snow)
elif moisture < DESERT_MOISTURE and temperature > WARM_THRESHOLD -> DESERT
elif moisture > FOREST_MOISTURE -> FOREST
else -> PLAINS (grassland)
```

## 9. Environment systems (`environment.py`)

Global, time-varying state advanced each tick:

- **Clock / day-night**: `time_of_day = (t mod DAY_LENGTH) / DAY_LENGTH`. Adds a diurnal
  temperature offset (cool at night, warm midday) and may modulate plant growth / activity.
- **Season**: `season = (t mod YEAR_LENGTH) / YEAR_LENGTH`. Adds a seasonal temperature
  offset and scales plant growth (winter = low growth) and thirst rate (summer = higher).
- **Weather**: simple stochastic state (e.g. clear / rain / heat) with smooth transitions
  (Markov or noise-driven). Rain → temporary moisture/growth boost; heat → faster thirst.
  Keep minimal in v1.
- **Temperature query**: `temperature(cell, t) = static_temp[cell] + season_offset(t) +
  diurnal_offset(t) + weather_offset(t)`. Used by perception (idx 22) and metabolism.
- **Nutrients**: per-cell soil nutrient pool [0,1]; depleted by plant growth & grazing,
  regenerates slowly. Drives overgrazing / competition dynamics.

## 10. Entities & species

SoA store in `entities.py`: parallel NumPy arrays sized to a max-capacity pool, plus an
`alive` mask and a `species` id array. Suggested per-entity fields:

```
pos_x, pos_y          float
heading_x, heading_y  float   # for movement momentum / exploration
energy, hunger, thirst, health   float [0,1]-ish
age                   float
sex                   int {0,1}
species               int {SHEEP, FOX}
genome                float[N_GENES]   # see §11
repro_cooldown        float
alive                 bool
```

**Vegetation as a per-cell field** (`vegetation.py`), NOT thousands of plant entities —
cheaper, and "food" perception just samples the field. Sheep/fox are pooled SoA entities.
(Decide this early; it shapes how perception's "nearest food" works.)

Species roles:
- **PLANT / vegetation**: stationary food field; regrows; value ∝ local
  nutrients/moisture/season.
- **SHEEP**: herbivore; eats vegetation; prey of fox; needs water; full genome.
- **FOX**: predator; eats sheep; needs water; full genome.

## 11. Genome (`genome.py`)

Fixed-length float gene vector per entity. **Heritable** genes (clamped to species ranges):

| gene | effect |
|---|---|
| max_speed | top movement speed |
| sensory_range | perception radius (the local-vision trait) |
| metabolism_rate | energy burn rate (trade-off vs capability) |
| size | energy capacity, food value when eaten, slight speed penalty |
| max_age | lifespan |
| repro_threshold | energy required to breed |
| (fox) aggression / (sheep) flee_distance | optional behavioral genes |

**Non-heritable**: `sex` (random 50/50 at birth), and all live state.

**Mutation**: per gene, with prob `MUTATION_RATE`, add `gauss(0, MUTATION_STRENGTH)`;
clamp to species range.

**Crossover** (sexual): child gene = uniform random pick from parent A or B per gene
(uniform crossover), then mutate. (Averaging is an alternative; per-gene pick preserves
more variance — recommended.)

## 12. Perception (`perception.py`) — LOCAL ONLY, per-species grids

Builds, **per species**, an `Observation` (egocentric grids `(N, C, K, K)` + scalars
`(N, SCALAR_DIM)`, §7.2) each tick, and returns `(obs_by_species, idx)` with the global alive
ordering. Window geometry (`R`, `K`, the distance stencil, and circular eye-masks per integer
radius) and the zero-padded static fields are precomputed once in `__init__`, so each agent's
`K×K` window is a plain sliding-window slice times its eye-mask — no per-agent index math.

For each species' agents:
1. **Scalars**: internal state (hunger/thirst/energy/health/age/sex), own-cell temperature,
   global clock/season, the agent's `sensory_range`, and `diet`.
2. **Field channels** (`terrain`, `water`): windowed slices of the padded static fields.
3. **Food + entity channels** (species-specific):
   - Sheep `food` = the windowed **vegetation field**; the brain later picks the *best grass
     cell* in range. Sheep also scatter in-range `threat` (foxes) and `mate` cells.
   - Fox `food` = in-range **exposed** sheep scattered into the window (sheep in cover are
     invisible — the refuge); fox also scatters `mate` cells. No `threat` channel.
   Entity scatters use per-species spatial grids; **cells beyond the agent's own
   `sensory_range` or off-world stay zero.** Never fall back to a global search.

Notes:
- "Threat" scatter is skipped entirely when no foxes are alive; juveniles skip the (adult-
  only) mate query — both big savings in a young-skewed population.
- "Mate" = in-range conspecifics of the opposite sex that are adult.
- Keep the build vectorized where possible; per-species grids keep it near O(N).

## 13. RuleBrain logic (`brain.py`) — the hardcoded v1 behavior

Vectorized priority arbitration, run **per species** over that species' observation. Since a
rule brain can't convolve, it first **decodes** the grid channels into the simple targets
below — `nearest_in_channel` (nearest present cell → nearest threat/mate/water/prey) and
`best_in_channel` (highest-value cell, mild distance pull → best grass patch). The explore-
heading RNG is drawn **once over the global ordering** so partitioning by species doesn't
change the run. Highest applicable priority wins; output a heading + the relevant gate.

```
for each agent (vectorized):
  if threat.present:                      # 1. SURVIVAL: flee
      heading = away_from(threat)
  elif most_urgent_need_satisfiable():    # 2. NEEDS: drink/eat
      need = argmax(hunger, thirst)       #    pick the more urgent need
      target = water if need==thirst else food
      if target.present:
          if adjacent(target): gate(drink/eat) = 1
          else: heading = toward(target)
      else:
          heading = explore()             # need exists but nothing in sight -> search
  elif can_reproduce() and mate.present:  # 3. REPRODUCE (adult, fed, watered, cooldown ok)
      if adjacent(mate): gate(reproduce) = 1
      else: heading = toward(mate)
  else:                                    # 4. EXPLORE (load-bearing)
      heading = explore()                  # directed wander: keep momentum + small noise
```

- `explore()` = continue current heading + small random turn (momentum so it doesn't
  jitter); reflect off impassable water / map edges.
- This brain never "chooses" death; metabolism kills (§14).
- This whole function is throwaway — its job is only to produce believable behavior and
  exercise the contract.

## 14. Systems detail

- **movement.py**: `heading * speed(genome,size) * dt` → new position. Block movement into
  ocean and into impassable elevation (high mountain optional); freshwater is passable
  (they drink at edges). Terrain speed cost (slope/biome) scales effective speed. Update
  `heading` arrays for next tick's momentum.
- **consumption.py**:
  - Sheep eat: if `eat` gate and on/adjacent to vegetation → transfer food value to
    energy, reduce hunger, deplete that cell's vegetation + nutrients.
  - Fox predation: if `eat`/attack gate and adjacent to a sheep → kill sheep (prob or
    deterministic), fox gains energy from sheep `size`.
  - Drink: if `drink` gate and adjacent to freshwater → thirst → 0.
- **metabolism.py** (per dt): `energy -= base_burn + move_cost·speed·size`;
  `hunger += hunger_rate`; `thirst += thirst_rate · heat_factor(temperature)`;
  if hunger high → energy/health drain; if thirst high → health drain; `age += dt`;
  death if `health<=0` or `energy<=0` or `age>max_age` (rising prob near max_age).
  Mark `alive=False`; free the slot.
- **reproduction.py**: candidates = adult + `energy>repro_threshold` + cooldown elapsed +
  hunger/thirst below limits + a perceived opposite-sex same-species adult adjacent. Pair
  them deterministically; female (or either) spawns `LITTER_SIZE` offspring near her;
  child genome = crossover(parentA, parentB) + mutate; both parents pay `REPRO_COST`
  energy and enter cooldown. Respect a per-species population cap.
- **vegetation.py**: each land cell's vegetation grows toward a carrying capacity set by
  local nutrients × moisture × season × temperature suitability; growth consumes
  nutrients; grazed cells regrow over time. No growth on water / extreme cells.

## 15. Determinism & RNG — two independent seeds

- **World seed** (`cfg.world.seed`) drives world generation **only** — terrain noise *and*
  hydrology (rivers get a generator derived solely from the world seed, never the run RNG).
  Same world seed ⇒ identical map, regardless of the run seed.
- **Run seed** (`cfg.seed`) drives all stochastic *dynamics*. A single
  `numpy.random.Generator` is created in `Config.make_rng()` and **passed into** every system
  that needs randomness. **No global `np.random`, no module-level RNG.** A run seed of `None`
  makes `make_rng` draw a fresh random seed and **record it back** onto `cfg.seed` (so each
  run differs but stays reproducible after the fact — the entry points print it).
- `make_config(world_seed=…, seed=…)` sets the two independently.
- Fixed `dt`. Deterministic iteration order (by slot index).
- Same world seed + same config + same run seed ⇒ identical run; same world seed + a
  different run seed ⇒ a different run on the *same* world (essential for headless experiments
  and reproducible evolution studies).

## 16. Rendering (`render/viewer.py`)

- Arcade `Window`/`View`. Holds a `Simulation` instance; calls `sim.step(dt)` inside
  `on_update`. **Observer only — never mutates sim state.**
- World drawn as a **background texture built once** from the field/biome arrays (color
  per biome; blue freshwater vs darker ocean; sand beaches). Rebuild only if world changes.
- Entities drawn via a batched `SpriteList` (sheep vs fox color/shape; size ∝ genome size).
- HUD: tick, populations per species, clock/season/weather.
- Controls: pause; speed (sim steps per frame); zoom/pan; vegetation overlay; season
  fast-forward/pause; manual spawn. **Left-click-to-inspect** rings an animal and renders its
  live egocentric perception grids (the channels for its species) in a panel — reads
  `sim.last_obs`, never mutates the sim.

```
SpeciesConfig (per species): init_count, gene_ranges{max_speed, sensory_range,
                metabolism_rate, size, max_age, repro_threshold},
                maturity_age, repro_cost, repro_cooldown, litter_size,
                hunger_rate, thirst_rate, base_burn, population_cap,
                mutation_rate, mutation_strength
SimConfig:      dt, grid_cell_size (≈ max sensory_range), max_entities, log_every
```

Calibration warning: local perception + gendered mating makes populations **fragile** — if
initial density is too low or sensory_range too small relative to map size, populations
collapse before breeding. Tune density / sensory_range / map size together first.

Predator–prey balance is especially delicate. The failure mode is the predator (fox) going
extinct at a prey trough, after which the prey explodes to its cap. Keeping foxes persistent
depends on three coupled knobs — the **prey-refuge size** (`World.cover`, ~30% of land is the
sweet spot; bigger starves foxes, smaller lets them over-crop and crash), the **Type III
half-saturation** (`hunt_halfsat`; do not over-lower it), and a **leaner fox metabolism**
(`base_burn`/`move_cost`/`hunger_rate` ~⅓ below the prey's so foxes survive lean periods).
The last is very sensitive. See `CLAUDE.md` "Calibration notes" for the full reasoning and
always re-run `run_experiment.py` (a few seeds, ≥6k ticks) after retuning.

## 20. Forward-compat notes (do NOT implement in v1 — just don't preclude)

- The `brain.decide(obs_by_species, idx)->act` contract + global batched `BrainSystem` is the
  seam for neural brains (MLP → LSTM → CNN retina → GNN). The per-species `(N, C, K, K)`
  perception grids are already a CNN's channel-stack input; keep act a plain matrix.
- **Local perception is deliberate**: it makes future **memory** (LSTM, replay-as-"sleep")
  meaningful — the agent will learn to remember where water/mates were and decide to stay
  or leave. v1 just leaves blind-time as exploration.
- `sensory_range` as a gene is the first hook for evolvable perception; later, evolvable
  **morphology** (body plan in genome, per-individual obs/act shapes via GNN-over-body-
  graph) is the research-hard endgame.
- Two future learning loops to keep separate + toggleable: within-lifetime RL, and
  across-generation neuroevolution (running both = Baldwin effect).
